Análise e Projeto de Sistemas (APS)

Compreensão dos Requisitos do Cliente

APS ensina os alunos a interagir com os clientes para compreender suas necessidades e requisitos para o sistema de software a ser desenvolvido. Isso inclui a coleta e análise de requisitos funcionais e não funcionais.

Modelagem de Sistemas

Os alunos aprendem técnicas de modelagem para representar os diferentes aspectos de um sistema de software, incluindo modelos de casos de uso, diagramas de classes, diagramas de sequência e diagramas de atividades.

Arquitetura de Software

APS aborda conceitos de arquitetura de software, ensinando aos alunos como projetar a estrutura global de um sistema de software, incluindo a divisão em componentes, a definição de interfaces e a organização dos módulos.

Metodologias de Desenvolvimento

Os alunos são introduzidos a diferentes metodologias de desenvolvimento de software, como o modelo cascata, o modelo iterativo e incremental, e o desenvolvimento ágil. Eles aprendem sobre as vantagens e desvantagens de cada abordagem e como aplicá-las no contexto de um projeto real.

Prototipagem e Validação

APS ensina os alunos a criar protótipos de software para validar os requisitos com os clientes e usuários finais. Isso ajuda a garantir que o sistema atenda às expectativas e necessidades do cliente antes da implementação completa.

Documentação e Padronização

Os alunos aprendem a documentar o processo de análise e projeto de sistemas de forma clara e concisa. Isso inclui a criação de documentos como especificações de requisitos, documentos de design técnico e manuais de usuário.

Testes e Qualidade de Software

APS aborda técnicas de teste de software e garantia de qualidade, ensinando aos alunos como planejar, projetar e executar testes para verificar a funcionalidade, desempenho e segurança do sistema.

Gestão de Projetos de Software

Os alunos são introduzidos aos princípios básicos de gestão de projetos de software, incluindo o planejamento, alocação de recursos, controle de prazos e custos, e gerenciamento de riscos.
